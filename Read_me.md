제가 만든 프로젝트는 PC방 관리 프로그램입니다.

이 프로젝트는 C언어로 진행하였는데요, 
C언어를 선택한 이유는 적은 용량으로 로컬 환경에서 빠른 실행 속도를 구현하고 싶었기 때문입니다.

[순서도 설명]

프로젝트 구조를 레이어드 아키텍처 방식으로 설계하였습니다.
레이어드 아키텍처는 소프트웨어를 기능별로 수평적으로 계층으로 나누는 방식으로,
각 레이어가 하나의 책임만 담당하여 코드 유지보수성을 높입니다.

저는 총 3개의 레이어로 나누었습니다:
- UI 레이어: 화면 입출력을 통한 사용자 상호작용
- Core 레이어: 비즈니스 로직 (회원 인증, 시간 계산, 요금 계산)
- Storage 레이어: 데이터 저장 (CSV 파일 기반)

[UI 화면 예시]

비즈니스 로직은 Core 폴더 아래에 두었으며,
로그인 담당인 auth, 요금 담당인 billing, 시간 담당인 time으로 구성했습니다.

먼저 로그인 기능을 설명드리겠습니다.
로그인은 회원이 존재하고 잔여 시간이 있을 때만 가능합니다.
로그인 시 현재 시간을 Epoch Time으로 기록하고,
회원의 남은 시간 정보를 반환합니다.

로그아웃 시에는 로그인 시간을 조회하여
현재 시간에서 로그인 시간을 빼서 경과 시간을 계산하고,
잔여 시간에서 경과 시간을 차감하여 업데이트합니다.
또한 초과 사용 시간이 있으면 요금을 계산합니다.

요금 담당인 billing은 회원이 시간을 충전할 때
기존 잔여 시간에 충전한 시간을 더하여 저장합니다.

시간 담당인 time은 회원의 실시간 남은 시간을 조회하는 로직입니다.
로그인 중인 경우 Epoch Time을 사용하여 경과 시간을 실시간으로 계산하고,
저장된 시간에서 경과 시간을 차감하여 정확한 남은 시간을 반환합니다.

이번 프로젝트에서 가장 어려웠던 점은 C언어의 특성상 값 복사만 가능하여
파일에서 읽어온 구조체 데이터를 전달하는 과정이었습니다.
이를 Output Parameter Pattern 방식으로 해결했습니다.
구조체의 주소값을 파라미터로 받아 함수 내부에서 데이터를 채워넣는 방식으로,
메모리 효율성과 성능을 개선할 수 있었습니다.
